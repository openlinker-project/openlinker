Product detail — v5: same as v3, minus Category mapping and Total income

Identical to v3 (docked photo gallery + lightbox, expandable Listings using this stock rows on the merged Variants & stock table) with two sections removed per feedback — neither belongs in the Products/Inventory merge this mockup exists to validate:

1. Category mapping — removed. Taxonomy/attribute mapping to marketplace categories is a separate concern from ProductMaster-vs-InventoryMaster; keeping it here diluted the comparison.
2. Total income — removed. It's an Orders-context aggregate, not ProductMaster or InventoryMaster data, and v3 rendered it as a hardcoded figure with no "proposed" flag — misleading on a page whose whole point is showing only real, already-shipped data. The Price card that shared its row now stands alone.
3. Two clocks, named separately — the hero's timestamp is now Catalog updated, and a second Stock synced … badge sits next to the Variants & stock heading. ProductMaster and InventoryMaster run on independent sync jobs, so the two figures can legitimately differ by a few minutes — one shared "Updated" field used to hide that.
4. Listing rows are now links — the platform name opens that listing's own detail page inside OpenLinker; the small ↗ icon opens the live listing on Allegro/Erli itself, in a new tab.
Everything else — gallery, lightbox, Available/Variants/Listings KPIs, External IDs, the merged stock table and its per-row expand — is unchanged from v3. Click a variant row's chevron to see: row 1 opens with 2 listings, row 2 has 1, row 3 demonstrates the empty state.
